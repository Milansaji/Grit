Next-Generation Authorization

Protect your routes with microsecond-precision access control

JWT Protection Middleware

// Initialize the protection middleware
auth := grit.MongoProtected(jwtSecret)

// Apply to routes requiring authentication
r.Get("/protected/resource", auth(yourHandler))
r.Post("/secure/data", auth(dataHandler))
r.Delete("/admin/user", auth(deleteHandler))
  • Cryptographic Validation: Verifies JWT signature using industry-standard algorithms (HS256/RS256)
  • Expiration Enforcement: Automatically rejects expired tokens with proper HTTP status codes
  • Instant Rejection: Blocks unauthenticated requests in microseconds with 401 Unauthorized
  • Context Injection: Seamlessly extracts and injects user context for downstream handlers
  • Attack Prevention: Guards against token tampering, replay attacks, and timing attacks

Granular Permission Control System

// Define permission-based middleware
adminOnly := grit.RequirePermission(jwtSecret, "admin:all")
moderatorAccess := grit.RequirePermission(jwtSecret, "content:moderate")
writerAccess := grit.RequirePermission(jwtSecret, "posts:write")

// Apply layered security to critical routes
r.Delete("/user", auth(adminOnly(deleteUserHandler)))
r.Post("/admin/config", auth(adminOnly(configHandler)))
r.Put("/content/approve", auth(moderatorAccess(approveHandler)))
r.Post("/blog/create", auth(writerAccess(createBlogHandler)))
🎯 Elite Permission Architecture: The admin:all permission grants omnipotent access, bypassing all permission checks as a superuser role. Design hierarchical permission strings with namespace prefixes (users:write, content:delete, analytics:read) for surgical access control. Combine multiple permissions for role-based access patterns.

Zero-Config CRUD Operations

MongoDB CRUD Engine

Model Registration & Schema Definition

type Blog struct {
    ID        primitive.ObjectID `bson:"_id,omitempty" json:"id"`
    Title     string            `bson:"title" json:"title" validate:"required,min=3,max=200"`
    Content   string            `bson:"content" json:"content" validate:"required"`
    UserID    string            `bson:"user_id" json:"user_id"`
    CreatedAt time.Time         `bson:"created_at" json:"created_at"`
    UpdatedAt time.Time         `bson:"updated_at" json:"updated_at"`
    Tags      []string          `bson:"tags" json:"tags"`
    Published bool              `bson:"published" json:"published"`
}

// Register your model with automatic indexing
grit.RegisterModel("blogs", &Blog{})

Complete CRUD Route Architecture

// CREATE - Insert new documents with validation
r.Post("/blogs", auth(grit.MongoC("blogs")))

// READ ALL - Query with pagination, filtering & sorting
r.Get("/blogs", auth(grit.MongoR("blogs")))

// READ SINGLE - Retrieve by ObjectID with optimized lookup
r.Get("/blog", auth(grit.MongoGetByID("blogs")))

// UPDATE - Modify existing documents with conflict resolution
r.Put("/blog", auth(grit.MongoU("blogs")))

// DELETE - Remove documents with cascade options
r.Delete("/blog", auth(grit.MongoD("blogs")))
⚡ Performance Optimization: Built-in query optimization, automatic indexing on user_id fields, connection pooling, and intelligent caching layer. Handles millions of documents with sub-millisecond response times.

Critical Security Protocols

Non-negotiable security principles for production-grade applications

⚠️ The Iron Rule of Ownership

NEVER, under any circumstances, accept user_id from client requests.

The backend must cryptographically derive ownership from the authenticated JWT token. Accepting user identifiers from the frontend creates a catastrophic security vulnerability enabling privilege escalation, data manipulation, and unauthorized access to other users' sensitive information.

✅ Correct Security Implementation

// ✅ CORRECT: Backend cryptographically extracts user_id from JWT
func CreatePost(w http.ResponseWriter, r *http.Request) {
    // Extract verified user ID from authenticated JWT context
    userID := r.Context().Value("user_id").(string)
    
    var post Post
    json.NewDecoder(r.Body).Decode(&post)
    
    // Use cryptographically verified userID for ownership
    post.UserID = userID
    post.CreatedAt = time.Now()
    post.Save()
    
    json.NewEncoder(w).Encode(post)
}

// ❌ CRITICAL VULNERABILITY: Never accept user_id from client
func CreatePostVulnerable(w http.ResponseWriter, r *http.Request) {
    var req struct {
        UserID  string // SECURITY BREACH!
        Title   string
        Content string
    }
    json.NewDecoder(r.Body).Decode(&req)
    
    // Attacker can manipulate user_id to create posts as other users!
    // This is a CRITICAL security flaw
}
🛡️ Defense-in-Depth Strategy: Always validate JWT signatures, enforce token expiration, implement rate limiting, use HTTPS exclusively, sanitize all inputs, implement CORS policies, and maintain comprehensive audit logs for security compliance.
